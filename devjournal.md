Panther Millionaire
Development Journal

Team Members:
Lan Le Dang
Cristin Khor

Day 1:
We looked through the project topic choices and picked a game inspired by Who Wants to Be a Millionaire. We also went over the main requirements like sessions, forms, login and registration, leaderboard behavior, and responsive design.

Day 2:
The main pages for the project were planned out, including the homepage, register page, login page, lobby, game page, result page, leaderboard, logout, and team intro page. We also talked through the flow from registration to login to gameplay.

Day 3:
The focus was on registration and login. We made sure usernames and passwords would be checked properly and that both players would need separate accounts.

Day 4:
More testing was done on the login and registration flow to make sure registered accounts could actually be used again later. Once that started working, the project felt much more solid.

Day 5:
The question bank and main game setup came next. We chose cat trivia and split the questions into easy, medium, and hard sections.

Day 6:
The question bank was expanded so the game could support 15 total questions. The prize ladder was also connected so the full game flow started coming together.

Day 7:
Turn-based gameplay for two players took a lot of attention. We had to keep track of both players, their progress, and whose turn it was without breaking the match flow.

Day 8:
Lifelines and the walk-away option were added. We also figured out how penalties would affect the final score.

Day 9:
The result flow was tested with correct answers, wrong answers, turn endings, and walk-away behavior. We checked that the game always moved to the correct next step.

Day 10:
Then we moved our attention to the leaderboard. We made sure scores were recorded, sorted, and shown correctly, even when there was no active match.

Day 11:
Styling and responsiveness were the next main focus. The layout, buttons, spacing, and page consistency were improved so the project looked better on both desktop and mobile.

Day 12:
The Team Intro page was updated so it clearly showed the project leader, project title, short description, and each member’s role.

Day 13:
Code cleanup. Comments were improved, and leftover demo-account references were removed once real registration was working correctly.

Day 14:
The project was checked against both the rubric and the Topic 02 requirements. We reviewed the 15 levels, lifelines, walk-away option, turn-based flow, and leaderboard behavior.

Day 15:
The README, AI disclosure, and other project documents were put together. We also started organizing what would be needed for the final submission.

Day 16:
We reviewed the required documents and checked what was still missing. Presentation and video planning also started here.

Day 17:
Presentation prep continued with a focus on what code to show, what features to demo, and what examples to use for the Q&A.

Day 18:
A final review was done on the whole project. We checked that all pages worked and made sure the code, documents, and presentation materials matched the final version of the project.
