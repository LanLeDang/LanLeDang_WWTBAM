Sprint 1: Planning and Proposal
Sprint Goal
Pick the project idea, go over the requirements, and plan out the main pages and roles.

Work Completed:
Looked through the project topic choices
Chose Topic 02: Popular Game Show
Decided to make a Millionaire-inspired game
Assigned team roles
Planned the main pages and user flow
Started proposal planning and wireframe ideas

Sprint Review:
This sprint helped us figure out the direction of the project and what the final app needed to include. By the end of it, we had a clearer idea of the game flow and the main parts of the project.

Retrospective:
The planning stage was helpful, but the requirements were spread across a lot of different pages. Going forward, we needed to keep better notes so we would not miss anything important.

Sprint 2: Project Setup and Main Structure
Sprint Goal
Set up the project files and build the starting structure for the app.

Work Completed:
Created the main PHP files
Set up the folder structure
Built the base navigation flow
Planned how sessions and cookies would be used
Mapped out the register, login, game, result, and leaderboard pages

Sprint Review:
This sprint gave us the base structure we needed to start real development. The project was organized and ready for the main PHP features.

Retrospective:
This part went smoothly and gave us a good starting point. The next step was making sure the backend logic worked correctly.

Sprint 3: Registration, Login, and Session Setup
Sprint Goal
Build the login system and get the session flow working for two players.

Work Completed:
Added player registration
Added 2-player login flow
Built account checking logic
Started storing match data in sessions
Added redirects for protected pages

Sprint Review:
By the end of this sprint, registration and login were working and the app had the basic session setup needed for the match flow.

Retrospective:
This sprint made it clear that session structure mattered a lot. It took some trial and error to make sure both players could be handled in the same match.

Sprint 4: Gameplay Features:
Sprint Goal
Build the main Millionaire gameplay and make sure it matches the topic requirements.

Work Completed:
Added 15-question progression
Split the question bank into easy, medium, and hard
Added 2-player turn-based gameplay
Added lifelines: 50/50, hint, and pass
Added the walk-away option
Connected prize progression and scoring

Sprint Review:
This sprint was a big step forward because the project started feeling like a full game instead of just a set of pages. Most of the Topic 02 features were added here.

Retrospective:
This was one of the hardest parts of the project because so many things had to work together. We had to be careful with turn order, scoring, lifelines, and result flow.

Sprint 5: Testing, Debugging, and Styling:
Sprint Goal
Test the full flow, fix issues, and improve how the project looks.

Work Completed:
Tested registration and login flow
Tested correct and wrong answer behavior
Tested walk-away and lifelines
Tested leaderboard updates and sorting
Fixed score and session issues
Improved CSS styling and responsive layout
Cleaned up comments and code readability

Sprint Review:
The project became much more stable in this sprint. The testing helped catch problems that were easy to miss earlier, and the styling became more polished.

Retrospective:
This sprint showed how important testing really is. A lot of smaller issues only showed up after running the full project from start to finish multiple times.

Sprint 6: Final Documents and Presentation Prep:
Sprint Goal
Finish the required documents and get everything ready for submission and presentation.

Work Completed:

Prepared the Project Report
Prepared the Development Journal

Prepared the Sprint Log
Updated the Team Intro page
Reviewed Q&A and presentation requirements
Planned the code walkthrough and demo flow

Sprint Review:
This sprint helped pull everything together. The code, documents, and presentation materials all started matching the final version of the project.

Retrospective:
The final sprint showed that documentation and presentation prep take a lot more time than expected. It also helped us see how important it is for the code, docs, and speaking parts to all match.
